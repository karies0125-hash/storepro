StorePro
Настройка Google Firebase — один раз,
занимает около 7 минут
💡 Это делается один раз. После настройки просто открываете файл и работаете — данные автоматически сохраняются в Google Cloud.
1

Создать проект Firebase

›
  1. Откройте в новой вкладке: console.firebase.google.com
  2. Войдите в свой Google аккаунт
  3. Нажмите «Создать проект»
  4. Введите название: например storepro-magazin
  5. Отключите Google Analytics (нажмите переключатель → серый)
  6. Нажмите «Создать проект», подождите ~30 сек
  7. Нажмите «Продолжить»
☑ Готово — нажмите чтобы отметить
2

Включить вход через Google

›
  1. В левом меню: Build → Authentication
  2. Нажмите «Get started»
  3. Вкладка «Sign-in method»
  4. Найдите Google и нажмите на него
  5. Включите переключатель Enable (синий)
  6. В поле «Project support email» выберите ваш email
  7. Нажмите «Save»
☑ Готово — нажмите чтобы отметить
3

Создать базу данных Firestore

›
  1. В левом меню: Build → Firestore Database
  2. Нажмите «Create database»
  3. Выберите «Start in production mode» → «Next»
  4. Регион: europe-west3 → «Enable»
  5. Подождите ~1 минуту
  6. Перейдите на вкладку «Rules»
  7. Удалите весь текст и вставьте правила ниже:
rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { match /storepro_users/{userId}/{document=**} { allow read, write: if request.auth != null && request.auth.uid == userId; } } }
  1. Нажмите «Publish»
⚠️ Обязательно замените правила! Без этого данные не сохранятся.
☑ Готово — нажмите чтобы отметить
4

Получить ключи и вставить сюда

›
  1. В Firebase нажмите ⚙️ (шестерёнка) → «Project settings»
  2. Прокрутите вниз до «Your apps»
  3. Нажмите иконку </> (веб-приложение)
  4. Введите название storepro-web, нажмите «Register app»
  5. Скопируйте значения из блока firebaseConfig в поля ниже:
Заполните все 6 полей выше
5

Разрешить домен

›
  1. В Firebase: Authentication → Settings → Authorized domains
  2. Нажмите «Add domain»
  3. Введите адрес где открыт файл: …
  4. Нажмите «Add»
💡 Если файл открыт с компьютера (file://), добавьте localhost. Для постоянного использования лучше разместить файл на любом хостинге (GitHub Pages, Netlify — бесплатно).
